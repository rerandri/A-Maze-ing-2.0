# A-Maze-ing

## Description

**A-Maze-ing** is a maze generation project developed in Python as part of the 42 curriculum.
The main goal is to design and implement a system capable of generating valid mazes while applying clean software architecture, algorithmic reasoning, and team collaboration practices.

At a high level, the project:
- Builds a maze data structure from a configurable setup.
- Generates the maze using a selected generation algorithm.
- Optionally supports visualization or export depending on implementation choices.
- Emphasizes reusable components for future maze-based projects (solvers, games, pathfinding demos, etc.).

---

## Features

- Simultaneous maze generation and solving.
- Config-driven setup (size, seed, optional display/output options).
- Modular Python codebase with reusable logic.
- Team-oriented development workflow.
- Multiple maze generation algorithms (`DFS` (Depth-First Search), `BFS` (Breadth-First Search)).
- Display mode (ASCII).
- Configurable randomness seed for reproducibility.
- Export to file / load from file.

### Advanced features implemented (Bonus):
- Extra interactive commands: /bench, /info, /clear, etc.
- Custom pattern via config: PATTERN='string'
- Custom maze colors: /setcolors or /sc
- Animation delay: /delay <seconds>
- Play mode: /play or /p

### Using the MazeGenerator module (reusable)

```python
from mazegen import MazeGenerator

# Create a 20x15 maze
maze = MazeGenerator(
    width=20,
    height=15,
    entry=(0, 0),
    exit=(19, 14),
    seed=42,          # reproducible
    perfect=False,    # Pac-Man mode (loops)
)

# Generate the maze
maze.generate()

# Solve it (BFS)
solution = maze.solve()  # list of "N", "E", "S", "W" directions

# Export as hex lines (output file format)
hex_lines = maze.to_hex_lines()

# Inspect individual cells
cell_mask = maze.get_cell(5, 3)          # bitmask 0-15
has_north = maze.has_wall(5, 3, maze.NORTH)
```

## Config File Structure

Example (`config.txt`):

```txt
# ---- SIZE ----
WIDTH=50
HEIGHT=50

# ---- ENTRY / EXIT ----
ENTRY=29,1
EXIT=1,29

# ---- OUTPUT ----
OUTPUT_FILE=maze.txt

# ---- OPTIONAL ----
PERFECT=true        # true = perfect maze (no loops), false = Pac-Man mode
SEED=42             # reproducibility

# ---- BONUS ----
ANIMATION=ON
DELAY=0.03
PATTERN=Hello
```

### Field Reference

- `maze.width` *(int)*: Number of columns.
- `maze.height` *(int)*: Number of rows.
- `maze.seed` *(int | None)*: Random seed for reproducibility.
- `maze.perfect` *(bool)*: Perfect (no loops) or imperfect maze.
- `maze.entry` *(tuple[int, int])*: Entry coordinates.
- `maze.exit` *(tuple[int, int])*: Exit coordinates.
- `maze.to_hex_lines()` *() -> list[str]*: Export grid as hex strings.
- `maze.get_cell(x, y)` *(int)*: Wall bitmask of a cell.
- `maze.has_wall(x, y, direction)` *(bool)*: Check a wall direction.

## Maze Generation Algorithm

### Chosen Algorithm: Recursive Backtracking (Depth-First Search)

The maze is generated by:
1. Starting from the entry cell.
2. Randomly visiting an unvisited neighbour.
3. Removing the wall between the current and next cell.
4. Recursively continuing until a dead end.
5. Backtracking to the latest cell with unvisited neighbours.

When ``PERFECT=False`` (Pac-Man mode), the generator opens the four corners
and the centre to degree ≥ 2, then carves at least 2 independent loops so a
chased player always has an alternative route.

### Why this algorithm

We chose Recursive Backtracking because it:
- Is simple and robust to implement.
- Produces “natural-looking” mazes with long corridors.
- Performs efficiently for typical project sizes.
- Is easy to explain, debug, and extend in a pedagogical context.

## Reusable Parts of the Code

The `mazegen` package is designed as a standalone module using only the Python
standard library (``random``, ``sys``, ``collections.deque``).  It can be
installed via pip and imported in any future project:

```bash
# Install build
pip install build
python3 -m build

# Install from the built wheel
pip install dist/mazegen-1.0.0-py3-none-any.whl
```

```python
from mazegen import MazeGenerator

# Instantiate
maze = MazeGenerator(
    width=20, height=15,
    entry=(0, 0), exit=(19, 14),
    seed=42, perfect=False,
)

# Generate
maze.generate()

# Solve (BFS) – returns list of "N", "E", "S", "W" directions
solution = maze.solve()

# Export hex grid
hex_lines = maze.to_hex_lines()   # list[str], one row per line

# Access the grid
cell_mask = maze.get_cell(x, y)   # int 0-15
has_north = maze.has_wall(x, y, maze.NORTH)
```

### Components designed for reuse

- **MazeGenerator class**: Complete maze generation, solving, and export in one file.
- **Grid model**: 2-D list of wall bitmasks, accessible via ``get_cell()``.
- **Wall utilities**: ``has_wall()``, ``_remove_wall()``, direction constants (``NORTH``, ``EAST``, ``SOUTH``, ``WEST``).
- **Config loader/validator**: ``parser/`` module for reading ``KEY=VALUE`` files.
- **Renderer/exporter**: ``display/ascii_renderer.py`` for terminal visualization.
